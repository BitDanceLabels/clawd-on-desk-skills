### X Post 2 — Thread 7 tweet (technical deep dive)
> 1/ Why your AI coding agent feels lonely:
> Terminal is silent. You don't know if Claude Code is thinking or stuck. Permission requests get buried under your browser. Multi-session = chaos.
>
> Built Bumbee to fix this. Thread on architecture 🧵
>
> 2/ Hook architecture
> Bumbee runs an HTTP server on 127.0.0.1:23333. Claude Code's command hooks fire-and-forget POST {state, session_id, event}. State machine resolves multi-session priority.
>
> [Diagram]
>
> 3/ Codex CLI doesn't support hooks on Windows (hardcoded off). So we poll JSONL logs in ~/.codex/sessions/ instead. ~1.5s latency vs ~150ms hook.
>
> 4/ Permission bubble = blocking HTTP hook
> Claude Code sends PermissionRequest, Bumbee responds {behavior: "allow"|"deny"} after user clicks bubble. 600s timeout. If user answers in terminal first, bubble auto-dismisses.
>
> 5/ Multi-window architecture
> 2 Electron windows: render window (transparent, click-through) + hit window (transparent shape, focusable). Solves Windows drag-drop bug from layered focus.
>
> 6/ State priority
> error(8) > notification(7) > sweeping(6) > attention(5) > juggling(4) > working(3) > thinking(2) > idle(1) > sleeping(0)
>
> Min display time prevents flicker.
>
> 7/ Code is MIT. Avatar Store + Team Dashboard are paid. Roadmap: Live2D, voice, mobile companion.
>
> Star: github.com/BitDanceLabels/clawd-on-desk-skills
> Try: bumbee.dev
